Imakefile modifications for KSFoundation and to sense EPIC releases

Make defines for EPIC_RELEASE in the Imakefile to use in a psd (and GERequired.e) to be less sensitive to upgrades

The EPIC preprocessor

  • converts .e files to .host.c and .tgt.c. Flags for this preprocessing goes in ADD_E_INCLUDES
  • understands %ifdef instead of #ifdef for the C preprocessor. However, there is no %if, only %ifdef/%endif.

To make a pulse sequence compatible with muliple EPIC releases, we create here one define for the EPIC preprocessor and one define for the C preprocessor (example here for release 25x):

  • EPIC_RELEASE_IS_25x (EPIC preprocessor). Allows %ifdef EPIC_RELEASE_IS_25x
  • EPIC_RELEASE = 25 (C preprocessor). Allow #if EPIC_RELEASE >= 25

These can be used in the sequence to handle quirks (e.g. change of # of arguments) in some functions, or new/obsolete functions, between releases. Hopefully, all release specific stuff can be placed in GERequired.e

To figure out which EPIC release that is currently set and put this number in RELEASE_NUM, one can try different ESE paths and extract this number from the path name (are there better ways?) 

TMP01 = $(patsubst /ESE_HD%_V01,%,        $(TOP))
...
RELEASE_NUM = $(patsubst /ESE_RX%.0_R02,%, $(TMP17))

PATCHTMP01 = $(patsubst /ESE_DV24_0_V0%,%, $(TOP))
...
PATCH_NUM = $(patsubst /ESE_RX27.0_R0%,%,  $(PATCHTMP07))

One can then use RELEASE_NUM and PATCH_NUM to make the following EPIC_** defines 

ADD_E_INCLUDES = -DEPIC_RELEASE_IS_$(RELEASE_NUM)x
ADD_CFLAGS_HH = -DEPIC_RELEASE=$(RELEASE_NUM) -DEPIC_PATCHNUM=$(PATCH_NUM)
ADD_CFLAGS_HS = -DEPIC_RELEASE=$(RELEASE_NUM) -DEPIC_PATCHNUM=$(PATCH_NUM)
ADD_CFLAGS_TH = -DEPIC_RELEASE=$(RELEASE_NUM) -DEPIC_PATCHNUM=$(PATCH_NUM)
ADD_CFLAGS_TS = -DEPIC_RELEASE=$(RELEASE_NUM) -DEPIC_PATCHNUM=$(PATCH_NUM)

Modify the Imakefile to use KSFoundation

KSFOUNDATIONPATH = ../KSFoundation

ADD_CFLAGS_HH = -I../$(KSFOUNDATIONPATH)
ADD_CFLAGS_HS = -I../$(KSFOUNDATIONPATH)
ADD_CFLAGS_TH = -I../$(KSFOUNDATIONPATH)
ADD_CFLAGS_TS = -I../$(KSFOUNDATIONPATH)

ADD_LINKFLAGS_HH = -L../$(KSFOUNDATIONPATH)/host
ADD_LINKFLAGS_HS = -L../$(KSFOUNDATIONPATH)/sim_linux
ADD_LINKFLAGS_TH = -L../$(KSFOUNDATIONPATH)/tgt
ADD_LINKFLAGS_TS = -L../$(KSFOUNDATIONPATH)/sim_linux

ADD_LIBS_HH = -lKSFoundation_hh
ADD_LIBS_HS = -lKSFoundation_hs
ADD_LIBS_TH = -lKSFoundation_th
ADD_LIBS_TS = -lKSFoundation_ts

Since RX27, GE has added -O2 and -Wfloat-equal to the compilation

-O2 makes compiling take about 5 min. To avoid this, and the float comparison complaining, the following has been added to the end of the Imakefile for each sequence: 

CFLAGS2_HS += -O0 -Wno-float-equal
CFLAGS2_HH += -O0 -Wno-float-equal
CFLAGS2_TS += -O0 -Wno-float-equal
CFLAGS2_TH += -O0 -Wno-float-equal

Since RX27.0_R02, TGT_MGD does not compile (and is not needed)

Premier systems (RX27) don't use TGT_MGD but TGT_ICE. Strangely, TGT_MGD is still compiled while forcing very old C conformance using a compiler different from RX27.0_R01. To compile RX27.0_R02 each pulse sequence's Imakefile now has SKIP_BUILD (taken from common.mk) to avoid TGT_MGD for RX27.0_R02 only: 

ifeq ($(SKIP_BUILD), FALSE)
    BuildingEpicPsd()
endif
